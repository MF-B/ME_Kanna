# 接口规范

## 消息命名约定
- **`evt_`** (Event): 通知/报告 — "告诉你发生了什么 / 当前状态是什么"
- **`cmd_`** (Command): 指令 — "请你执行一个动作"
- 所有 WS 消息统一信封: `{"type": "...", "data": {...}}`

---

## 1. 采集端 ↔ 后端 (WebSocket · `/ws/collector`)

### 1.1 上报流 (Lua → Backend)

| 消息类型 | 触发方式 | 说明 |
|---------|---------|------|
| `evt_tick` | 每 1 秒轮询 | 物品库存 + CPU 状态 + 能源 + 存储 |
| `evt_crafting` | `os.pullEvent` 异步 | 合成事件回调 (开始/完成/错误) |
| `evt_production` | 海龟每周期上报 | 工厂产量流水 (delta) |
| `evt_craftables` | 后端下发指令触发 | 上报所有可合成物品列表 (带库存数量) |

### 1.2 下发流 (Backend → Lua)

| 消息类型 | 说明 |
|---------|------|
| `cmd_craft_item` | 下发合成任务 `{item, amount}` |
| `cmd_set_priority` | 更新 Priority 名单 `{items: [...]}` |
| `cmd_set_routine` | 更新 Routine 名单 `{items: [...]}` |
| `cmd_scan_craftables` | 触发可合成物品全量扫描 (无 payload) |

---

## 2. 前端 ↔ 后端

### 2.1 动态状态流 (WebSocket · `/ws/frontend`)

后端接收采集端原始数据，聚合加工后推送给前端。**前端不感知采集端的数据格式**。

| 消息类型 | 方向 | 说明 |
|---------|------|------|
| `evt_sync` | Backend → Frontend | 聚合后的完整状态快照 (items/cpus/energy/storage/craftingStatus) |
| `evt_crafting` | Backend → Frontend | 转发合成事件 (用于 toast 通知等) |

> 注：两条 WS 连接完全独立，消息命名不冲突。

### 2.2 HTTP 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/itemInfo/:itemId` | 获取物品静态资源 (Icon、本地化名称) |
| GET/POST | `/api/autocraft/tasks` | 自动合成任务列表 / 创建任务 |
| DELETE | `/api/autocraft/tasks/:itemId` | 删除自动合成任务 |
| PATCH | `/api/autocraft/tasks/:itemId` | 修改任务 (启用/停用、调整阈值) |
| GET | `/api/craftables` | 获取当前可合成物品列表 |
| POST | `/api/craftables/scan` | 触发后端要求 Lua 立刻重新扫描可合成物品列表 |

### 2.3 静态资源

| 路径 | 说明 |
|------|------|
| `/lua/*` | Lua 脚本 OTA 下载 |
| `/icons/*` | 物品图标静态文件 |
