# 系统边界
本项目是一个针对 Minecraft (我的世界) Applied Energistics 2 (AE2) 网络的状态监控与任务下发平台。

## 系统职责与边界
- **数据采集端 (Collector / Lua)**:
  - 职责：
    - 接收后端 WS 下发的 **Priority 名单** 与 **Routine 名单**。
    - 在 1 秒轮询周期中，**全量查询** Priority 名单，**时间分片轮转** Routine 名单，保护游戏 TPS。
    - 打包上报：物品库存、CPU 状态、能源统计、存储统计。
    - 监听 `os.pullEvent("me_crafting")`，异步上报合成事件。
    - 接收并执行后端的合成指令。
    - 支持 OTA 启动脚本：通过 HTTP 从后端拉取最新 Lua 代码。
  - 边界：**不做数据聚合与计算**，不做全量遍历查询，不做持久化。
  - 注：海龟（Meter 设备）作为独立采集端，负责监测工厂产能并上报产量流水。

- **后端 (Backend / Go)**:
  - 职责：
    - 维持两条独立的 WebSocket 连接：
      - `/ws/collector`：与采集端双向通信，接收原始数据 + 下发指令。
      - `/ws/frontend`：向前端推送聚合后的状态（`evt_sync`）。
    - 将上报状态缓存在内存中（`AE2Network` 聚合结构）。
    - 数据聚合：将采集端原始数据加工后推送前端，包括：
      - 合成状态推导：结合 `AutoCraftRule` + `Inventory` + `ActiveJobs` 产出五态 `craftingStatus` 指示灯。
      - 工厂产能计算：接收海龟上报的 delta，计算生产速率。
    - 管理并下发 Watchlist（Priority / Routine 双名单）。
    - 提供 HTTP API：物品静态字典、自动合成任务 CRUD、Lua 脚本 OTA。
    - 支持阈值自动合成：库存低于阈值时自动触发合成指令。
  - 边界：当前阶段**暂不持久化**（不连接 MySQL/SQLite），不渲染前端页面。

- **前端 (Frontend)**:
  - 职责：
    - 通过 HTTP 字典 + WS 动态流的双 Store 模式，展示 AE2 库存、能源、存储容量、CPU 状态和工厂产能。
    - 从 `evt_sync` 的 `craftingStatus` 字段直接渲染合成指示灯，无需自行推导。
    - 提供合成任务下发、自动补货阈值设置的交互 UI。
    - 支持 i18n（中文 / English），物品名称本地化。
  - 边界：**绝不直接与采集端通信**，所有数据均通过后端中转。
