# 数据结构

## 消息命名约定
- **`evt_`** (Event): 通知/报告
- **`cmd_`** (Command): 指令

---

## 采集端 → 后端 (原始态)
除 `os.pullEvent("me_crafting")` 外，其他数据都使用轮询发送。

### 统一消息信封
```json
{
  "type": "message_type",
  "data": {}
}
```

### evt_tick (心跳包, 每 1 秒)
```json
{
  "type": "evt_tick",
  "data": {
     "items": [
       {"name": "minecraft:iron_ingot", "count": 1024},
       {"name": "minecraft:raw_iron", "count": 50}
     ],
     "cpus": [
        {
            "coProcessors": 1,
            "storage": 65536,
            "craftingJob": null
        },
        {
            "coProcessors": 1,
            "storage": 65536,
            "craftingJob": {
                "count": 16,
                "name": "ae2:certus_quartz_dust"
            }
        }
     ],
     "energy": {
       "stored": 100000,
       "capacity": 1600000,
       "usage": 25.5,
       "input": 130.0
     },
     "storage": {
       "itemTotal": 16384,
       "itemUsed": 8192,
       "fluidTotal": 4096,
       "fluidUsed": 1024
     }
  }
}
```

### evt_crafting (异步合成事件)
```json
{
  "type": "evt_crafting",
  "data": {
    "jobId": 12,
    "isError": false,
    "message": "CRAFTING_STARTED"
  }
}
```


## 后端 → 前端 (聚合态)

### evt_sync (状态推送)
后端接收采集端原始数据，聚合加工后推送。
```json
{
  "type": "evt_sync",
  "data": {
    "items": [
      {"name": "minecraft:iron_ingot", "count": 1024},
      {"name": "minecraft:raw_iron", "count": 50}
    ],
    "cpus": [
      {"coProcessors": 1, "storage": 65536, "craftingJob": null},
      {"coProcessors": 1, "storage": 65536, "craftingJob": {"name": "ae2:certus_quartz_dust", "count": 16}}
    ],
    "energy": {"stored": 100000, "capacity": 1600000, "usage": 25.5, "input": 130.0},
    "storage": {"itemTotal": 16384, "itemUsed": 8192, "fluidTotal": 4096, "fluidUsed": 1024},
    "craftingStatus": {
      "minecraft:iron_ingot": "idle",
      "minecraft:diamond": "crafting",
      "minecraft:redstone": "disabled"
    }
  }
}
```

#### craftingStatus 五态说明
仅包含 Priority 名单中配置了 AutoCraftRule 的物品。

| 状态 | 条件 | 含义 |
|------|------|------|
| `disabled` | `Rule.IsActive == false` | 规则存在但未启用 |
| `idle` | `Rule.IsActive == true` 且库存充足 | 补货已开启，暂不需要合成 |
| `pending` | `Rule.IsActive == true` 且库存不足，尚无 Job | 等待合成 |
| `crafting` | 存在对应 `JobInfo` 且 `Status == crafting` | AE2 正在合成 |
| `error` | 存在对应 `JobInfo` 且 `Status == error` | 合成失败 |

## 后端 - 存储态
```Go
// ========================
// 物品
// ========================

// 物品静态信息 (字典, 缓存在内存中, 按需懒加载)
type ItemInfo struct {
	ItemId string `json:"itemId"`
	Name   string `json:"name"`
	Icon   string `json:"icon"`
}

// 物品动态信息 (随每次 evt_tick 更新)
type ItemState struct {
	Count int64 `json:"count"`
}

// ========================
// CPU & 合成任务
// ========================

// CPU 上的合成任务 (来自 AE2 原始数据)
type CraftingJob struct {
	Name  string `json:"name"`
	Count int    `json:"count"`
}

// CPU 状态 (来自 AE2 原始数据)
type CPUState struct {
	Storage      int          `json:"storage"`
	CoProcessors int          `json:"coProcessors"`
	Job          *CraftingJob `json:"job"` // nil = 空闲
}

// 平台下发的合成任务追踪
type JobInfo struct {
	JobID  int    `json:"jobId"`
	ItemId string `json:"itemId"`
	Count  int    `json:"count"`
	Status string `json:"status"` // pending, crafting, error
}

// ========================
// 能源 & 存储
// ========================

type EnergyStats struct {
	Stored   float64 `json:"stored"`
	Capacity float64 `json:"capacity"`
	Usage    float64 `json:"usage"`
	Input    float64 `json:"input"`
}

type StorageStats struct {
	ItemTotal  int64 `json:"itemTotal"`
	ItemUsed   int64 `json:"itemUsed"`
	FluidTotal int64 `json:"fluidTotal"`
	FluidUsed  int64 `json:"fluidUsed"`
}

// ========================
// 工厂产能
// ========================

type FactoryState struct {
	FactoryId   string             `json:"factoryId"`
	FactoryName string             `json:"factoryName"`
	Items       map[string]float64 `json:"items"` // itemId -> 生产速率 (个/分钟)
	IsActive    bool               `json:"isActive"`
	LastUpdated int64              `json:"lastUpdated"`
}

// ========================
// 自动合成
// ========================

type AutoCraftRule struct {
	ItemId       string `json:"itemId"`
	MinThreshold int64  `json:"minThreshold"` // 低于此值触发合成
	MaxThreshold int64  `json:"maxThreshold"` // 合成到此数量
	IsActive     bool   `json:"isActive"`
}

// ========================
// 聚合根
// ========================

type AE2Network struct {
	mu sync.RWMutex

	// 监控名单
	PriorityWatchlist map[string]bool // 前端视口 + 活跃任务, 全量高频查
	RoutineWatchlist  []string        // 补货名单, Lua 分片轮转查

	// 物品
	Registry  map[string]ItemInfo  // 静态字典 (懒加载)
	Inventory map[string]ItemState // 动态库存

	// CPU & 合成
	CPUs       []CPUState       // AE2 CPU 状态快照
	ActiveJobs map[int]*JobInfo // 平台活跃任务

	// 系统状态
	Energy  EnergyStats
	Storage StorageStats

	// 工厂
	Factories map[string]*FactoryState

	// 自动合成规则
	AutoCraftRules map[string]*AutoCraftRule
}
```

## 前端 - 展示态
前端采用双 Store 模式进行状态管理：
1. **字典库 (Static Store)**: 通过 HTTP `GET /api/itemInfo/:itemId` 按需懒加载物品静态信息 (Icon、本地化名称)。
2. **状态库 (Dynamic Store)**: 通过 WebSocket 接收后端 `evt_sync` 推送的聚合数据。
3. **视图层 (View)**: 利用前端框架的计算属性 (Computed / Selectors)，在本地将两者结合，渲染完整的监控界面。`craftingStatus` 直接映射为指示灯颜色。
4. **国际化 (i18n)**: 支持中文 / English，物品名称通过静态字典本地化。
